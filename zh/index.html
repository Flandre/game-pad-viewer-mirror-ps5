Info & About

关于这个工具

我之所以开始制作这个工具是因为我发现显示游戏手柄输入的所有其他选择都太麻烦了， 或者根本无法根据我的喜好进行自定义。一段时间后，我意识到其他人可以从我制作的工具中受益， 并且自从我更新和扩展原始功能以来。我仅从Xbox 360外观开始， 此后，我添加了多个外观，创建自定义外观，缩放，输入延迟和许多其他功能的功能。

值得庆幸的是，由于Marcin Wichary's的Gamepad.js和Tester.js库，所有这些都是可能的，我已经为自己的目的对其进行了修改。

色度键安装

  1. 创建新的窗口捕获源
  2. 选择gamepad viewer窗口
  3. 添加 "色度值"
  4. 点击“颜色”旁边的“选择”按钮，然后使用吸管工具选择 页面上的棕色背景
  5. 将“相似度”和“混合度”设置为所需的值
  6. 启用“子区域”
  7. 单击“选择区域”，然后将区域框缩小到所需大小
  8. 点击“确定”，您就完成了！

使用浏览器源安装

  1. 前往URL生成器创建自己的游戏手柄URL
  2. 单击生成的预览URL以将其复制
  3. 在OBS中，创建一个新的浏览器源
  4. 将复制的URL粘贴到浏览器源的“ URL”文本框中
  5. 宽度和高度需要进行调整才能获得正确的值， 因此只需简单地从源已经设置的默认值向上或向下工作
  6. 点击“确定”，您就完成了！

经常问的问题

这到底是什么？

这是一个基于Web的工具，用于直观地表示游戏手柄的输入。 您基本上可以在屏幕上看到使用游戏手柄所做的事情。 通常，人们会使用这样的工具在屏幕上显示流或录制视频的控制器输入。

如何设置为录制/流式传输？

只需按照信息页面上的步骤进行操作，您就可以立即开始使用。

如何制作自定义皮肤？

好吧，您需要具有CSS的一些基本知识，以及如何从头开始制作CSS，但是我有一个示例文件可以为您提供帮助。请务必每隔一段时间检查一次，尽管我确实进行了更改以与网站上的更新保持一致。

我可以编辑皮肤而不仅仅是制作自定义皮肤吗？

是的！您可以使用editcss=URL参数指向正确的格式的CSS文件，它将编辑当前选定的皮肤。 这是一个小示例编辑文件 只是为了说明可以做什么。

参数文档

本节包含每个URL参数的文档。如果您想生成自己的自定义网址，请访问此页面 此页面。

玩家编号

https://gamepadviewer.com/?p=1
  • N/A
  • 1
  • 1,2,3,4
  • 设置此参数将启动“紧凑模式”，该模式会删除背景和UI元素，因此仅显示控制器

玩家编号参数是选择在紧凑模式下查看哪个连接的控制器的主要方法。

皮肤ID

https://gamepadviewer.com/?s=1
  • N/A
  • 1
  • 0,1,2,3,4,5
  • 将ID设置为除上述值以外的任何值，则默认将皮肤设置为“ 1”，即“ Xbox One”皮肤
  • 如果计划使用皮肤ID参数，则需要设置玩家编号(p=)

设置皮肤ID是选择要在紧凑模式下显示的游戏手柄的唯一方法。如果您想使用自定义外观，则需要阅读以下css=参数。

强度计

https://gamepadviewer.com/?smeter=1
  • N/A
  • 0
  • 0,1
  • 除“ 1”以外的任何值均会禁用此设置

启用此设置后，触发器的视觉表示将是强度计，而不是不透明度级别。

缩放系数

https://gamepadviewer.com/?sc=1.2
  • N/A
  • 1
  • 任何数字，包括十进制值
  • 负值只会翻转皮肤，同时也会负面缩放皮肤

启用此设置可让您使用整数或十进制值缩放游戏手柄的大小。

死区半径

https://gamepadviewer.com/?dz=0.3
  • N/A
  • .25
  • 0到1之间的任何数字，包括十进制值
  • 将此值设置为1或更大的值基本上会禁止摇杆的移动

如果您的旧游戏手柄遇到摇杆漂移，或者仅仅是不想显示游戏手柄产生的每一个微小动作。

Input Delay

https://gamepadviewer.com/?delay=2000
  • 毫秒
  • 0
  • 任何正数

如果您从PS4捕获游戏玩法，并且将控制器连接到计算机和 控制台，此设置适合您。只需在控制台中输入从控制台到PC的流延迟 毫秒，您的输入将被同步，而不是提前同步。

旋转停止

https://gamepadviewer.com/?rot=450
  • 角度
  • 120
  • 任何正数
  • 这个数字是总旋转的一半。例如。最大旋转900度的将具有450度的旋转停止。
  • 死区值将会影响此设置。推荐设置dz=0。

此设置主要用于使指杆代表轮子，并且可以选择设置 它们的最大旋转度。

自定义CSS URL

https://gamepadviewer.com/?css=https://gist.githubusercontent.com/mrmcpowned/d61ff0d8d0c57cc207d8/raw/minimal%2520xbox.css
  • N/A
  • None
  • 指向格式正确的CSS文件的任何URL
  • 此参数禁用id=参数
  • 所选皮肤的CSS类将替换为.custom类

使用此参数可以将Gamepad Viewer的设计完全更改为您自己的或某人已共享的设计。

编辑CSS URL

https://gamepadviewer.com/?editcss=https://gist.githubusercontent.com/mrmcpowned/a787fc4f5307b7c008fb/raw/edit%2520example.css
  • N/A
  • None
  • 指向格式正确的CSS文件的任何URL
  • .edit 类适用于所有游戏手柄

类似于css=参数，您可以编辑当前选定外观的CSS，而不必制作全新的外观。

Version 0.7.0 - March 8, 2017

  • New Skin: There's a sorta-new skin in the available list: The white PS3 controller. Edits are courtesy of Million Lights
  • OBS Studio & Issues: A recent update to OBS Studio has updated the browser source, and in doing so has partially broken the Gamepad Viewer. A workaround exists so you can continue using the Gamepad Viewer until I can find out where it is exactly the bug that's causing this issue is, and fix it (but that will most likely take some time, cause school). The steps to fix are as follows: 1. Double click the Browser Source which has your Gamepad Viewer custom URL 2. Scroll down and click "Clear Cache" 3. Click "Cancel", not "OK" and not the "X" button at the top right 4. Make DOUBLE sure you click "Cancel" else you'll have to repeat steps 2 and 3.
  • I'm not dead: I'm still alive, even though school has me beaten down. I still answer emails though, so if you have any issues just send one of those, or if you're more of a cool kid like me, join me in my Discord server.

Version 0.7.0 - August 17, 2016

  • Patreon: Yup, this thing's on Patreon. If you know someone who'd be willing to support, feel free to point 'em in the right direction.
  • New Skin: The long-awaited and much-requested GameCube skin is finally out! Sorry I kinda took too long with this one, it's just that I wanted to get some stuff together and ship this with some other things that needed to be done, as follows...
  • GameCube controller Fix: So, if you own a GameCube controller and MayFlash adapter, you'd get some really weird values for the triggers and sticks, even if you tried remapping. Now, there's new remapping setting which can help you solve that! Just watch the update video to find out how to do just that!
  • Remapping Axis/Button exclusion: Another small but useful update to button remapping is the ability to disable a button or axes' ability to be selected when remapping inputs. This is typically useful for when you have an out of control axis that changes values rapidly and don't want it interfering with remapping. To cancel an input's mapping ability, simply right click it's blue box on the mapping page after a player is selected. To re-enable the input, simply right click the red box.
  • DPAD POV Mapping: Another common issue was that of DPADs that show up as a single axis input, or a Point-Of-View Hat Switch. With the new "DPAD" remapping option, just select the DPAD's axis, fill in the requested values, and the DPAD should be fixed!
  • Remapping feedback: When I was asking someone to test the new remapping options, i found that they had a hard time of telling if something worked or if it failed. To fix that, should a mapping failed, you'll see a persistent error message in red above the appropriate mapping, and if it succeeds, a green success message which fades away.
  • Remapping UI Populated via URL: Before when you would access exported remapping settings via URL, you couldn't edit them or customize them. Now, when accessing the exported mappings via URL, the remapping page will populate with the appropriate settings.
  • Input hints in Generate Page: Now, before entering information into the text boxes you see input hints. Hopefully this helps with people who are unsure of what to place in those inputs.
  • New Generate Page options: You can now disable stick curving and change the stick offset as well.

Author's Note - Yeah, it's a tad complicated, but I'll be making a video soon on how to set mappings up easily. Here's a sort of mini tutorial on how to do it. Then I realized people load skins insecurely from their own sites, to which I hurriedly changed the setting. So, if your stylesheet or any assets are loaded via HTTP, access the site from HTTP, else use HTTPS.
  • To name, or not to name: Another sort of unimportant-but-still-interesting-I-kinda-suppose-maybe updates was changing the attribute used to select buttons and axes from name to data-name which has it fall more inline with HTML5's semantics. Riveting stuff, I know.
  • Addendum on Remapping: It should be noted that you can't half remap an axis, like say you only want to map the positive or only the negative end. If you try to do that, the mapping is just straight up ignored.
